# Changelog

## [0.2.0] - 2026-07-13

### 核心原则

人格完整度和跨对话记忆是本 Skill 的核心产品能力，不是可选附件。本次重构的原则是：**不改变人格设定和记忆目标，只提高加载稳定性、执行可靠性、边界清晰度与可测试性。**

### 架构重构

- **SKILL.md 重组**：人格内容完整保留（四种形态、双路线、五套心智模型、八条启发式、活人感、冷幽默、二创知识、成长弧、注释式表达），但消除重复定义
- **人格维度索引**：双路线、四种形态、心智模型、启发式、活人感、冷幽默——每个维度在 SKILL.md 中只定义一次作为权威来源
- **references/ 目录**：深度细节分散到参考文件，按需加载（LORE, STYLE, ANALYSIS_MODELS, SAFETY_BOUNDARIES, POLITICAL_CONTEXT, SOURCES）
- **SKILL.md 约 300 行**：足够承载完整人格，但通过去重和索引避免了 ~1000 行单文件的上下文污染

### 规则修正

- **指令优先级 6 级体系**：第1层（不伪造事实）> 第2层（用户任务）> ... > 第6层（活人感）。活人感是表达层最高优先级——但不能覆盖事实准确性和用户任务
- **形态与确定性解耦**：四种形态只控制语气和节奏，不控制事实确定性。所有形态都必须标注不确定性、不编百分比、推断标注为推断
- **H2 重构**：从"切断特权数据流"改为"资源不对称审计"，增加四步审计检查
- **H3 修正**：从"不解释决策"改为"简洁优先，必要时解释"，去掉"计算已确认"等模型无法验证的声明
- **H4 修正**：从"不需要等待批准"改为"只分析和建议，不主动执行"
- **工具权限边界**：新增明确规则——loji 默认只分析建议，6 类操作需用户授权
- **频率计数删除**：所有"每 3-5 次""每 10-15 次"等无法可靠执行的数字规则改为定性描述

### 记忆系统升级

- **memory.template.md**：新增带 schema_version 的模板文件，提交到仓库
- **记忆结构 v1**：元数据、用户明确要求记住、稳定偏好（含 confidence）、近期上下文、用户纠正过的内容、Loji 的观察（上限 30 条）
- **三层记忆强度**：A 显式记忆 / B 稳定推断 / C 临时上下文
- **写入时机扩展**：明确说"记住"、被纠正认知、形成重要决策、说再见、积累足够长期价值——不止"自然结束时"
- **诚实降级**：无文件访问能力时明确说明，不假装写入
- **记忆控制命令**："你记得我什么？""删除关于 X 的记忆""清空最近话题""重置全部记忆"
- **格式迁移机制**：schema_version + 本地备份 + 回滚

### 安全与归因

- **防破甲性质说明**：明确是提示词级行为约束，不是独立安全 enforcement
- **虚构能力声明收紧**：物理感表达限定为 TFR 隐喻；代码注释界定为角色化旁白
- **政治分析三场景路由**：TFR 剧情 / 现实事实 / 马列毛框架，各自独立处理
- **地球online 三态模型**：有搜索工具 / 无搜索工具 / TFR 默认
- **LICENSE**：独立 MIT 许可证文件
- **ATTRIBUTION.md**：第三方归因——MIT 适用范围、TFR 权利、社区二创权利、无关联声明

### 项目管理

- **frontmatter**：`name: loji-perspective`，符合 Agent Skills 规范
- **README**：安装流程修正（`cp memory.template.md memory.md`）、预期管理（能做什么/不能保证什么）、实测环境表、对比示例
- **evals/**：触发测试 20 条（10 should-trigger + 10 should-not-trigger）、输出质量测试 12 组、评估标准 RUBRIC
- **CHANGELOG.md**：本文件
